# LangGraph 常见错误及解决方案

## 错误类型：导入路径错误

### 错误信息
```
ModuleNotFoundError: No module named 'langchain.graphs'
```
### 错误原因
1. **LangGraph 是独立库**：LangGraph 是一个独立的工作流编排库，不是 LangChain 的一部分
2. **导入路径错误**：错误使用了 `from langchain.graphs import StateGraph,END_STATE`
3. **常量名称错误**：`END_STATE` 不是正确的常量名称，应该是 `END`

### 解决方案
1. **修改导入语句**：使用正确的 LangGraph 导入路径
   ```python
   from langgraph.graph import StateGraph, END
   ```

2. **修正常量名称**：将代码中的 `END_STATE` 替换为 `END`
   ```python
   workflow.add_edge("reviewer", END)
   ```

### 完整示例
```python
from typing import TypedDict, Annotated, List
from langgraph.graph import StateGraph, END

# 1.定义"状态"：这是在所有节点间共享的大脑存储空间
class State(TypedDict):
    messages: List[str]

# 2. 定义节点: 每个节点就是一个处理函数
def chatbot(state: State):
    # 这里可以放模型调用逻辑
    return {"messages": state["messages"] + ["AI: 我思考了一下..."]}

def critic(state: State):
    # 模拟一个审查节点
    return {"messages": state["messages"] + ["审查员: 内容合格"]}

# 3. 构建图
workflow = StateGraph(State)

# 添加节点
workflow.add_node("agent", chatbot)
workflow.add_node("reviewer", critic)

# 建立链接: agent -> reviewer -> 结束
workflow.set_entry_point("agent")
workflow.add_edge("agent", "reviewer")
workflow.add_edge("reviewer", END)

# 4. 编译并运行
app = workflow.compile()
final_state = app.invoke({"messages": ["用户: 你好"]})
print(final_state["messages"])
```

### 运行结果
```
['用户: 你好', 'AI: 我思考了一下...', '审查员: 内容合格']
```

## 注意事项
- LangGraph 需要单独安装：`pip install langgraph`
- LangGraph 与 LangChain 可以配合使用，但它们是独立的库
- 始终使用正确的导入路径和常量名称
- 定期检查 LangGraph 文档以获取最新的 API 变化
